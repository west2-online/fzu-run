第一我要先劝退你 转专业不是退路 计算机 早已今非昔比 就业竞争压力大 如果你没有强大的决心要转专业 那很可能你会失败+挂科 为什么这么说呢 因为从决定转专业开始 你不仅要学习计算机编程语言 深入的了解一些算法 还要保证不挂科 如果挂科了就没有资格报名了(并且一个人只有一次转专业机会 关注一下往年的转专业资格 可能有变动) 一旦转专业失败可能会对你的学业有双重打击

第二 转专业会耗费你大量的时间 你是否真的愿意每天待在自习室学习 是否愿意一点一点的啃下来难懂的算法 按照我的情况给你做指导 每天都要待在自习室里刷题 一天真的很累 如果你没有这样的决心 还是早点放弃吧 转专业失败会给你带来巨大的挫败感

第三 算法不可能速成 需要一点一点的学习 沉淀 反思 总结 不可能考前速成 刚开始一道简单的“回文字符串判断”要卡2小时，提交10次才通过，挫败感会反复冲击你。如果只是“试试水” 不如把时间花在本专业 避免两边都落空

ok 现在你下定决心要转计算机学院了 接下来一步一步跟着我的脚步

正式开始(有取舍的阅读 后附图片)

一. 了解考试

  1. 考察内容: 大二转计算机考的是程序设计 考察编程语言的数据结构与算法 实话实说2025年考到了一道关于树的题目 但是难度较大 也没有人做出来 应该先把中心放在算法上 这些数据结构可以先放一放

  2. 考试难度: 难度接近PTA乙级

  3. 考试语言: 2025可以选择C++/C语言/Python/Java 考试没有自动补全 请在平常练习的时候不要使用自动补全

  4. 考试题型分值: 2025年是8道题(800分) 一道题有一个或者多个测试点(一个测试点5~100分) 实时提交 可以无限次提交 提交的次数不会影响成绩 可以实时看到所有题目的通过率

  5. 我作答情况(参考): 2025年AC(accept)了4道题 其余题目部分得分就已经是第二了 (最后一刻看到情况是这样的:有两道题通过人数为0(会有两三道超过PTA乙级难度是正常的 也没有人写出来) 一道题通过人数为5 一道题通过人数为1 其他题较多人通过) 所以在考试时不要慌 有两道特别难的不用担心 你做不出来 别人也是这样 不要影响自己 心态也时考察的一部分

  6. 提交界面: 2025年时直接在PTA的界面上写代码 不能用DEVC++ (往年是可以用的)

  7. 考试注意: 不能中途退出PTA界面(你一旦退出或者打开了其他界面老师会收到报警) 有防作弊系统 不要忘记带学生证和身份证 会发草稿纸 笔自己带

二. 小白怎么入门语言

编程语言推荐C++ 大部分人都会选择C++(C++兼容C语言 意思是C语言的语法再C++中都适用) 你可以先学C语言 再学C++ 但记住如果只会C语言是不行的 因为C++中的STL(Standard template library)是解题的关键 一定要学 不然很多题太耗时间了 你也会思路混乱 这个是重中之重

C语言推荐 翁恺的C语言编程课

C++推荐黑马程序员的课程 b站都可以找到

数据结构与算法推荐浙江大学陈越教授的<数据结构与算法>

接着你需要一个编写代码的工具DEVC++ 它是一个轻量化的代码编辑工具 安装时选默认配置即可 编写完代码ctrl+s保存 编译 运行 就在终端显示运行结果了 没有太多繁琐的功能 适合小白入门

看完之后尝试写一些简单的算法实现 比如交换两个数 数组求最大值 当然这些是完全达不到考试难度的 继续学习吧 学习完C语言就可以写PTA乙级题了 大概感受一下

三. C语言->C++->进阶算法

  • 已经学会了C语言了 就开始C++的学习 面向对象的知识并不是首要 因为这几年都没有考 不过可以学有余力再去学习 重要的是C++的一些库函数 STL库的熟练应用string vector 二维数组(2025考到了一般难度 你应该做出来) stack queue multiset set unordered_set map unordered_map(2025考到了) deque…他们的增删改查 时间效率 空间效率 怎么用这些容器优化算法 这个非常重要 没有STL容器别想在20min做出来

  • 主要:迭代器 sort函数 结构体排序 自定义排序 模拟算法 简单的贪心算法 字符串处理 哈希表(只会使用就ok 当然深入学习最好) 进制转化 素数判断(2025的第一道题 很简单) 最大公因数 最小公倍数 简单递归(不常考) 图形打印 线性查找 栈stack的使用 队列queue的使用

  • 学有余力(拉开差距)：二分查找 双指针 滑动窗口 动态规划 贪心算法 dfs bfs 前缀和优化 差分优化代码(这些算法会让你打开新世界的大门 但是进度太慢就不要学了)

  • 在写代码时注意：不要使用万能头文件 #include<bits/stdc++.h> 有时候PTA不能通过编译 这也是一个不好的习惯

四. 小白栽坑

  1. 禁止眼高手低.在学习语法的时候一定要上手写一写 不要好高骛远 觉得自己会了 语法不是靠背的 是要实战练习的 建议在学习过程中就开始刷题 基础不牢地动山摇

  2. 初期别怕“写得丑”.我第一次写“素数判断”用了30行代码，后来优化到10行，进步是靠一次次修改来的

  3. 重视编译错误：比如“忘记加分号”“数组越界”，这些基础错误在考试中会浪费大量时间，平时写代码就养成“写完先检查语法”的习惯

五. PTA常见提示(问GPT都可以轻松找到总结 但是不要太依赖AI 毕竟考试只有自己 没有AI陪你)

Compile Error (编译错误)

语法错误，代码无法被编译器识别，比如少分号、头文件缺失、变量未定义

  1. 先检查是否漏写分号、大括号配对；

  2. 确认使用的容器/函数对应的头文件已包含（如用string需加#include）

  3. 检查变量名是否拼写一致（比如“count”写成“cout”）

Runtime Error (运行时错误)

代码能编译，但运行时崩溃，常见原因：数组越界、除以零、指针异常

  1. 数组操作时确认下标范围（比如数组大小为n，下标别超过n-1）

  2. 除法运算前判断分母是否为0；3. 避免使用未初始化的指针（转专业机试少用指针，用vector替代更安全）

Wrong Answer (答案错误)

逻辑错误，输出结果与正确答案不符，是最常见的错误

  1. 检查测试用例是否考虑边界值（如n=0、n=1，输入为空）

  2. 确认变量类型是否匹配（比如用int存10^10的数，需改成long long）

  3. 浮点运算时用fabs(a-b)<1e-6替代直接比较a==b

Time Limit Exceeded (超时)

代码运行时间过长，超过题目限制，多因算法效率低

  1. 用STL容器优化（比如用unordered_map替代map，查找效率更高）

  2. 避免多层嵌套循环（比如O(n²)的算法改成O(n)）

  3. 减少不必要的输入输出操作（用scanf/printf替代cin/cout会更快）

Memory Limit Exceeded (内存超限)

代码占用内存过多，多因数组开太大或重复创建大量对象

  1. 数组按需开大小，避免固定开1e6以上的数组（用vector动态扩容）

  2. 重复使用变量，避免在循环内频繁创建大对象

  3. 及时释放无用的内存（C++中vector可通过clear()释放）

Presentation Error (格式错误)

输出格式不符，比如多空格、少换行、大小写错误

  1. 严格按题目要求输出分隔符（比如“用空格分隔”别用逗号）

  2. 确认每行末尾是否有多余空格（可在输出最后一个元素后不输出空格）

  3. 题目要求大写输出时别写成小写（比如“YES”别写成“yes”）

Output Limit Exceeded (输出超限)

输出内容过多，远超题目要求，多因循环逻辑错误

  1. 检查循环条件是否正确（比如把i<n写成i<=n导致多输出）

  2. 确认是否有多余的调试输出（比如测试时的cout<<"test"没删掉）

Non-Zero Exit Code (非零退出码)

程序异常退出，多与运行时错误类似，常见于数组越界或栈溢出

  1. 重点检查递归深度（递归次数别超过1e4，否则栈溢出，改用迭代实现）

  2. 按“运行时错误”的解决办法排查，优先检查数组和循环逻辑

六. 关于做题

  1. PTA乙级题目并不是按照难度排序 可以先做自己会的题目 刚开始可能比较吃力 一道题要做好久提交好多次 这是正常的 因为我也这样过来的

  2. 如果太难就先做PTA上<团体程序天梯赛>的题目 但是还是要转到PTA乙级题目上 毕竟这个难度较小 没有达到考试难度

  3. (如果我提到的这些你都能学会 并且PTA乙级所有题目都刷完 几乎每一道题都能在15min做出来 leetcode刷了100+题目 那我可以很负责的告诉你 你大概率会是第一或者第二名 因为我就是这样刷题的)

  4. 如果你起步较晚 一定一定要把PTA乙级题目刷完一遍 总结归纳题型 多做总结 其实考点也就专门几种

十. 是否降转

建议降转,给自己一个缓冲的机会 如果平转了你很有可能会一个学期考20门课程 太累了

我选择降转的原因: 以后规划做算法工程师 想再空余的一年自己练习算法 参加ACM 毕竟自己也想有更多的课余时间 毕竟靠技术吃饭 参加一些竞赛 保研什么的

十一. 如果你时间紧迫 先抓大头

  • STL容器：string（拼接、截取、查找）、vector（动态数组，增删改查）、map/unordered_map（统计频次，2025年考题）、stack/queue（模拟题常用）。重点记容器的常用函数，比如vector的push_back()、map的find()。

  • 基础算法：素数判断（2025年第一题，简单）、最大公约数（辗转相除法）、最小公倍数（两数乘积/GCD）、进制转换（10转2/8/16，除基取余法）、字符串处理（回文判断、大小写转换）。

  • 排序与查找：C++ sort()函数（必须会自定义排序规则，比如按字符串长度排、降序排）、线性查找（遍历数组/字符串）。

  • 模拟题：这是乙级核心题型，比如“模拟排队”“日期计算”，重点是把题目规则拆成步骤，用循环和条件判断实现

十二. 机试小技巧

  1. 机考的时候有些题目 直接cout<<[];输出答案就可以得到部分测试点的分数(我就是这样干的)

  2. 尽量早点到考场 我进考场了之后发现我的电脑是紫屏 校园网也翻吃了好久 乱七八糟的 还好最后登录了

  3. 开始作答之后页面弹出来小广告 我点了一下❌ 没想到直接跳转了 也是很无语了 最后老师来问我 她没有计较 反正就是做题的过程比较曲折吧 遇到这种情况直接找老师就行了
